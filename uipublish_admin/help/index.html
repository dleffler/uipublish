Introduction

Welcome to the UIPublish Web Publishing System. (UIPublish).  UIPublish enables you to easily publish and edit content on your website.  This User's Guide is designed to get you started quickly and provides information on how to do common tasks such as adding, editing or deleting content.

If you are eager to get started skip to " Getting Started". If you are interested in how UIPublish works, read on.

Note: As UIPublish can be easily customized, your webmaster may have customized some options of UIPublish. The examples and images below may not exactly match your website's installation of UIPublish. Please ask your webmaster for details.

How It Works

UIPublish is web-based system for managing dynamic content on a website. It is suitable for publishing articles, announcements, events, etc. It provides an administration interface for content managers and the front-end pages can be customized. The UIPublish system consists of three components.
  • A database back-end.
  • A control panel for administration
  • The front-end display

At the core of the system is a database that stores all the content you will enter into the system. You can use the control panel to add, modify, schedule, or delete content from the system. The front-end display component displays "live" content on your website.

Content is stored as an "item." There are two types of "items" -- " article items " and " event items ". Article items are for content such as news or press releases. In other words, items with a publication date. Event items are for content such as appointments, meetings, or workshops. In other words, items with an event date.

  Article items with a publication date that matches today's date or earlier are displayed on the website. Lists of article items are in reverse chronological order which means that the latest or newest item is displayed first.

  Event items with an event date that matches today's date or is in the future, are displayed as an upcoming event. Lists of event items are in chronological order which means that the list begins with the first or next upcoming item. Past events can also be displayed. These are listed in reverse chronological order which means that the most recent past event is displayed at the top of the list. 

Both types of items can be classified in to separate sections. For example, with UIPublish you can manage content in a "News" section as well as "Reports" section. Items can be listed on the main page of a website as well as a separate list for each section. Items can also be assigned a "Front Page" setting so that only items set to display there will appear on the main page of the website. Each item has a title, summary, content, and event/post date. Each item can also be assigned a category, an expiration or end date, an external link or URL to be used in lieu of the content, and may contain an "attachment" to allow a "PodCast" of a file kept in a directory on the filesystem.  Events may also include start and end times.  For more information see "Description of Options".

More detailed technical information about UIPublish is available at uipublish.sourceforge.net.

Getting Started

UIPublish Control Panel

To start using the UIPublish you will need to login to the UIPublish Control Panel. Depending on how it has been setup, you may need a username and password to access the control. You can get this information from your Webmaster.

UIPublish has the ability to organize content into different sections. This is setup by the Webmaster during the installation of UIPublish. On the main page of the Control Panel you will see one or more sections listed. Select a section to which you want to add an item. You should now see the Manage page which enables you to add, modify, reschedule, or delete items within that section.

Adding A New Item

To add a new item, simply select the "ADD" button adjacent to the text, "Add a new Item". You will then see a page titled "Add New Item...." with a form.  Fill out this form.

All items flagged with a red * must be filled out completely in order for the form to process correctly.  (For more information about the different options see "Description of Options."). Choose the default options for now. After you have filled out the form, choose the "Preview"

You should now see the preview page. This page is meant to enable you to preview the content you have entered. You can check the information you have entered for typos or other errors. If required information is missing you will be presented with a reminder to add the missing information. If all the required information has been filled out, you will see a button title  "Submit."  Select "Submit" to add the new item. If successful, you will see "Go back to the Manage Page." The information you entered is now added to the database but, depending on the options you have chosen, the item may or may not be displayed on the website. To check that you need to refer to the Manage Section page.

Manage Section

This page displays the items which have been entered into the system. Each item is listed in the table with a "Modify" button to its left and a "Delete" button to its right.  The background color of the table cell for each item indicates which items are "active" or visible on the frontend. Items for which Post Status , Post Date , and Expire Date are green will be actually online or visible on the website. The others are in the system but are not displayed on the website. If your webmaster has configured your website  to use the Front Page options then items  a dark blue square under Post Status will be displayed on the main page of the website.

The item that you just added is not yet online because it's "Post Status " is set to "Draft. " You can now modify the item to set the article's "Post Status " setting to "Publish ", (See "Modify an Item") By default all items are added with the post status set to " Draft " unless you specify otherwise.

Modify an Item

To modify an item, select the "Modify" button next to the item you would like to change. This will take you to the Modify form page with fields that are filled in with the information corresponding to the item you are modifying. Change whatever information needs to be changed. If this is a Draft item and you want to make it available online then change it's Post Status setting to " Publish ."  When you are finished, select the Preview button. If all the required information has been filled out, you will be presented with a preview page and the ability to Submit this information.  Select Submit to apply the changes to the item.

Delete an Item

Select the "Delete" button that is to the right of the item you wish to delete.  On the next page, select the Delete button shown.  If you do not wish to delete the item at that time, select No, do not delete.

Warning: Deleted items cannot be restored.

ADVANCED MANAGEMENT

UIPublish Control Panel

The UIPublish Control Panel acts as a master control center and provides a wealth of information about all article and event sections.

The Control Panel displays the numbers of items in each section including those items still in draft status, easily taking you to a list of draft items.  The Control Panel also allows you to quickly edit recent posts.

Manage Section

The Manage Section has several features to aid in finding items to edit or copy.

There is a Search feature to help you find items which contain a specific word or phrase in their title, summary, or content.  The Search filter may also be removed to display all the items in the section.

Most of the Columns can also be selected to sort both ascending and descending thereby grouping items.

The individual title can also be clicked to display the actual web page used to display that item.

Copy An Existing Item

Sometimes it's easier to use an existing item as a template for a new item.  To do this find the item you want to use and "Modify" it.

You notice a checkbox at the top of the page which allows you to create a copy instead of modifying this event.  If you do NOT check this box, you will end up modifying the original event.

Copy An Event Item to an Article

This feature allows you to use a new or existing event item and save it as a news item.  It's particularly handy since many events require a news item after the event has occurred.  In this case the event will no longer displayed, but the details of the event relate to the associated news item.

To copy an event to an article, use the associated button at the bottom of the preview page.  You must understand the event item will NOT be entered or updated using this option.  It is best to only use this when modifying existing events.

Optional Features

These are additional features that may be installed on your server.  Please contact your webmaster for details.

WYSIWYG Editor

UIPublish can be set up to use FCKeditor which allows editing the content entries as if using a traditional word processor.

Popup Calendar

UIPublish can be set up to use PopCalendarXP which allows a popup control for easily selecting dates and times.

Advanced File Management

UIPublish can be set up to use Kae's File Manager to upload and manage uploaded files and select files for Enclosures.  It contains many advanced features which also allow image or picture manipulation and other advanced file management features.  This option is typically installed as a plug-in for FCKeditor.

Web Calendar Integration

UIPublish can be set up to post events to a Web Calendar, though this is an older application.  If installed and configured, this will appear as an option on the event preview screen.  If selected, this item will be posted to the Web Calendar in addition to the UIPublish section.

Using The Built-IN File Manager

The built-in File Manager enables you to upload files to a particular directory. You can also delete uploaded files.
 

To upload a file, choose the "Browse" button. A dialog box should show up. Use it to locate the file you want to upload.

The filename will show up in the box. Choose "Upload" to upload the file. Depending on the file size, this may take a few minutes.


After the file has been uploaded it is listed in the File Manager along with its size and a link to delete the file. Warning: If you click the "Delete" link the file will be deleted without asking for a confirmation.

If you are having trouble uploading a file, check the file size of the file you are trying to upload. The File Manager is configured with a Max File Size limit. If the file size is larger than the limit it may not upload. Also, the File Manager has a limit on the total space used for uploaded files. If there is inadequate free space, the file may not upload. Ask your webmaster to increase the limits or delete files that are no longer needed. 

Description of  Options

Article Sections

Title

Required; the title of the item.

Publish Date

Required; allows you to specify when the item, if approved, will be online or appear on the site.

Expire Date

Optional; allows you to specify when the item will go offline.  If left blank, item will remain online until deleted.

Post Status

Required; if set to Draft, item will not go online until it is reviewed and set to Publish.

Front Page

Required; if set to Yes, item will appear on main page of website.

Section

Default; The section is set automatically when you add an item and you should not need to change it. However, if you need to you can move an item from one section to another you can using this menu.

Category

Ask your webmaster how to use this feature. Otherwise just leave it blank.

Summary

Optional; a short description of the content, usually used as a teaser on the main page of the website.

Content

Optional; the content of the item should be typed or pasted here. To start a new paragraph use the "Return" or "Enter" key on your keyboard. You can also use selected HTML tags. For allowed HTML tags see the "HTML" link adjacent to the content file or ask your webmaster.  You can also add URLs in the content field and they will be automatically hyperlinked. However, a URL that appears at the start of a new line may not be hyperlinked. If you are having problems getting a URL to hyperlink, add a space before the URL.

Link URL

Optional; if there is a related link to the item and you would rather have the user taken there instead of the content page, enter the Universal Resource Locator (URL or web address) here.

Enclosure

Optional; to attach a file for use as a Podcast or appear as an attachment in a newsfeed, enter the URL or web address here.

Event Sections

Most of the options for the event sections are the same as the article sections expect for Event Time(s) Date(s)

Start Time

Optional; it sets the time the event begins.

Event Date

Required; it sets the date of the event.

End Time

Optional; it sets the time the event ends.

End Date

Optional, but Recommended; it sets the date the event will end, meaning the event should no longer be listed as an upcoming event. If the date is in the future, then the item is active and listed on the website. If it is in the past , then the item may not be listed on the website or may be listed under the "Past Events" list if it has been setup by the webmaster. If this entry is blank, the event will remain active.
